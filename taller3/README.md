El patrón Factory se aplica aquí para la exportación de documentos: el cliente pide un exportador indicando el formato (PDF, Word o HTML) y recibe una implementación concreta sin conocer las clases reales. `DocumentExporterFactory.createExporter(DocumentFormat.PDF)` devuelve un `PdfExporter`, pero el cliente trabaja solo con la interfaz `DocumentExporter`. Así se evita instanciar `PdfExporter`, `WordExporter` o `HtmlExporter` directamente; la decisión de qué clase usar está centralizada en la factory y el cliente queda desacoplado de las implementaciones.
